Academic Projects
2013 [Final Year Project]
ARmax - A Cross-Platform Augmented Reality Development Kit

Abstract:
Augmented Reality has become a third eye to the day-to-day user providing a whole new dimension to the computing world. With the technological development of the mobile industry Augmented Reality concepts have soon been adapted to mobile devices.

However, due to the complexity of Augmented Reality features and the variation of mobile platforms, developers tend to use various native Augmented Reality Software Development Kits and Application Programming Interfaces when developing mobile Augmented Reality applications. Conversely, when it comes to the project level most of the clients require their applications to be run on a number of mobile platforms. Using different native Software Development Kits and Application Programming Interfaces is not an optimal solution because it forces a developer to do a substantial amount of re-work there by causing a lot of problems to the developer.

ARmax, a cross platform Augmented Reality Tool Kit is being proposed, which helps to develop Augmented Reality based applications and experiences for mobile platforms. ARmax is a JavaScript based Augmented Reality Tool Kit (framework) and it is introduced to create Augmented Reality objects/experiences to the modern web browsers which supports new technologies like HTML5 and CSS3.

In order to demonstrate the features of the AR framework a reference application was developed. A critical testing and evaluation was carried out to ensure the efficiency and ease of use of the ARmax framework as well as the prototype, where the testing verified 89.25% success rate. In addition, the evaluation feedback of the domain experts verified the novelty of the proposed solution and the benefits such as ease of use and platform independency.

Subject Descriptors:
H.5.1 – Multimedia Information System
H.5.2 – User Interface

Keywords:
Augmented Reality, Hypertext Navigation and Maps, Input Devices and Strategies, Interaction Styles, Screen Design

2012 [ECW M 604 Advanced Web Technology - Assignment]

A web application assignment implemented using mainly Spring MVC, Java, and MySql. For information please refer to the AWT Assignment 1 Report and AWT Assignment 2 Report.

Background: On this module, over the two assessments, you will design and implement a web application for the Human Resources (HR) division of a large company.

Overview: HR deals with the administration of all matters relating to employees, and the proposed system will help keep track of all employees (who number roughly 300,000) in the organization. For this coursework, you will be using the employee sample database available from the MySQL website at http://dev.mysql.com/doc/employee/en/index.html. The script could be modified be used to load data to any database system. You need to make sure you have a database installation you can use (either on your own machine or on the university server) and download and install the same data from the link above.

IMPORTANT: Moreover, there are two important non-functional requirements for the application. These two points will be important for the second coursework, the implementation, but also need to be borne in mind during coursework 1, the design. They are:

1. It should use a REST API wherever possible
2. AJAX style of communication between browser and server.

REQUIREMENT: Your tasks for this coursework relate to the design of the web application. Your tasks are as follows:

1. Decide on and describe a sensible set of requirements. (Estimated at 2 pages) You should look at the database schema for the sample database and decide what requirements it will sensibly support. You should also decide who is going to use the web application (there may, for example, be more than one type of user), and ask what requirements each type of user may require. For example, everyone in the organization may want to know which department someone works for, but not everyone should be able to find out what salary they earn.
2. The requirements should be visualized as a set of draft screens or mockups showing how the application will appear to users. (Estimated at 3-4pages)
Do not put just one mock-up screen on one page, you can include several in the same sheet.
3. Based on the requirements, create an outline structure of an MVC based web application, featuring controllers, views, and models. The structure should be presented using a UML class diagram. (Estimated at2 pages)
4. Design the process of user authentication.
Show how users should be authenticated and in what contexts. For example, should everyone who uses the site be authenticated or only those who have privileges to see certain information or carry out certain actions?
5. How are users to be authenticated? Do you require any extra tables in the database? (Estimated at 2 pages)

Your aim in this Coursework is to implement, using PHP/CodeIgniter, the design for the HR application in your submission for Coursework 1 (or the design in the sample answer provided for Coursework 1 if you so choose).

In addition to any requirements in your design (or the sample design), you must also implement the following:

1. A search facility, using an HTML form, that will allow anyone, without having to authenticate themselves, to search for employees by combination of employee name, department name, and title. For example, it should allow a search by department name and title, or by last name and department, or just by last name. The HTML form must conform to the following requirements:

a. the action attribute of the form tag must be //index.php/find/findemp (i.e. there must be a ‘find’ controller with a method called ‘findemp’)
b. the input fields of the form must be named ‘lastname’, ‘dept’,‘manager’,‘title’ etc. (all lower-case)
c. The request method on the form must be ‘GET’

An example input form will be made available and the link emailed out shortly. It is important that you follow these instructions precisely because this form will be used for automated testing of your application. If you are unclear what to do for this requirement, please ask.

2. The user logins should include some specification of role - user or admin - so that only people logging in as admins can see people’s salaries.

You must also write a short report, of no more than three pages, providing a critical evaluation of your implementation. Discuss what you think are its good points and bad points; how did it differ from your ideas at the design stage; how could it be improved?
